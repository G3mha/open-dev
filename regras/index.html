Avaliação

Como vocês podem ver na aula de apresentação, essa disciplina permite uma liberdade enorme e a maneira que isto está organizado é via gameficação. Cada participante cria um perfil no repositório e ganha skills (e XP) ao entregar as possíveis atividades. Toda entrega é feita via PR para o repositório da disciplina.

A composição da nota é um misto da quantidade total de XP recebida por enviar tarefas, a quantidade de XP obtida por membros de seu grupo e alguns critérios de barreira. A tabela abaixo resume essas condições:

Conceito Pontuação Critérios extras
I ---- Não fez alguma das atividades de tutorial
D ---- Completou todas atividades de aula, mas não chegou no C
C 50XP
  • 1 x Contribuição de código +
  • Tradução aceita! ou Contribuição de documentação
    		•
    B 90XP
  • Contribuição de código aceita
    • A 150XP
    A+ 200XP

    Durante o semestre as pontuações da tabela acima podem passar por revisões para baixo, mas nunca para cima.

    Skills disponíveis

    Cada skill se traduz em evidências para um dos objetivos de aprendizagem da disciplina. O conjunto de todas as skills feitas no semestre indica a proficiência dos estudantes em cada um dos objetivos.

    Tutorial

    Nesta parte inicial são feitas atividades preparatórias que apoiam o desenvolvimento das skills de Código, Documentação e Tradução e Comunidade.

    id Nome Descrição XP
    1 Primeiros passos Se adicionou no repositório da disciplina. 2
    2 Explorando o entorno de um projeto Pesquisou sobre a história e a comunidade de pessoas e serviços ao redor de um projeto. 3
    3 Projeto não documentado Recebeu um sistema sem documentação e o pôs para funcionar. 5
    4 Projeto Profissional Reformou um código mal feito e criou uma página web para ele. 5
    5 Testado e Aprovado Criou testes de unidade e de interface para o servidor de desafios. 5
    6 Pacote Python Criou um pacote python instalável via pip. 3

    8 Tradução básica Usou ferramentas de tradução para traduzir e localizar um pequeno programa em linha de comando. 2
    9 Minha primeira contribuição Primeiro PR enviado corrigindo um bug simples. 5
    10 Pesquisa de temas FLOSS Fez uma apresentação longa envolvendo um tema recente e relevante relacionado a software livre. 5

    Código

    Toda atividade que resulte em uma contribuição de código com projetos de software aberto é aceita na disciplina.

    id Nome Descrição XP
    21 Contribuição de código PR enviado corrigindo um bug. 7
    22 Contribuição aceita! Um PR não trivial foi aceito por um projeto externo. 13
    23 Contribuição (simples) aceita! Um PR simples (com mudanças triviais ou pequenas refatorações de código) foi aceito por um projeto externo. 3
    24 Bug Report Reportou um bug em um repositório e outro usuário conseguiu reproduzí-lo. 5
    25 Feature Request Fez um pedido de nova funcionalidade a um software. 2
    26 Projeto INSPER Fez mais contribuições a um projeto educacional do INSPER 5

    Comunidade

    Um projeto de software também é definido pelas interações de todas as pessoas ao seu redor: usuários, desenvolvedores e tradutores. Nesta categoria entram contribuições não-técnicas que beneficiam essas pessoas.

    id Nome Descrição XP
    40 Issue indireta Conversas em uma issue ou PR fizeram outros usuários abrirem issues relatando novos bugs/melhorias. 3
    41 Beta tester Ao reproduzir um bug você descobriu que ele já tinha sido corrigido e causou o fechamento de uma issue. 3
    42 Tradução aceita! Fez uma tradução que foi aceita em um software aberto. 5
    43 Novas línguas Adicionou suporte a traduções em um software que não era traduzível. 10
    44 Revisor Revisou pequenos erros na documentação de um projeto e teve suas modificações aceitas. 2
    45 Contribuição de documentação Fechou uma issue marcada como docs 7
    46 Pitch de projeto Apresentou um Pitch de projeto. 0
    47 Pitch aceito! Enviou contribuição para um projeto apresentado em sala. 5
    48 Sugestão aceita! Outro aluno enviou contribuição para o projeto que você fez Pitch. 3
    49 Replay de PR Apresentou (em 15 minutos) o processo de resolução de um bug na aula. 5

    Impacto

    Nesta categoria entram skills que representam o impacto que as atividades feitas no contexto da disciplina geraram tanto em um projeto de software como na comunidade a sua volta. Isto inclui,

    1. palestras dadas pelo alunos
    2. grande número de contribuições aceitas em um único projeto
    3. citação de trabalhos feitos pelo aluno em redes sociais do projeto
    4. releases contendo commits feitos no contexto da disciplina
    id Nome Descrição XP
    60 Palestrante Fez uma palestra sobre bibliotecas ou softwares livres. 20
    61 Helpdesk Teve uma resposta aceita em sites como StackOverflow ou em listas de email 4
    62 Contribuição publicada Uma contribuição sua foi incluída em um release. 10
    63 Convidado VIP Após alguma contribuição alguém do projeto te pediu para fazer uma tarefa. Após entregar a tarefa, adicione esta skill. 5
    64 3 PRs enviados 3 PRs enviados para o mesmo projeto 10
    65 3 PRs aceitos 3 PRs que você enviou para um projeto foram aceitos. 30
    66 Jabá Algo que você fez foi citado pelo projeto em redes sociais ou algum meio oficial. 10
    77 Hacktoberfest Participou da Hacktoberfest, enviando 3 PRs para projetos externos. 20
    78 Sprint de Tradução Elementary Participou do sprint de tradução com ao menos 40 strings traduzidas. 15
